## 一、项目技术特点总结

### 1.1 技术栈概览

| 层次       | 技术             | 版本      | 说明       |
| -------- | -------------- | ------- | -------- |
| **后端框架** | Spring Boot    | 2.5.15  | 核心框架     |
| **持久层**  | MyBatis        | -       | ORM框架    |
| **数据库**  | MySQL          | -       | 关系型数据库   |
| **缓存**   | Redis          | -       | 缓存加速     |
| **前端框架** | Vue.js         | 2.6.12  | Web管理端   |
| **移动端**  | uni-app + Vue3 | 3.0.0   | 跨平台移动应用  |
| **UI组件** | Element UI     | 2.15.13 | UI组件库    |
| **认证**   | JWT            | 0.9.1   | Token认证  |
| **二维码**  | Google ZXing   | 3.5.1   | 二维码生成与解析 |
| **报表**   | JasperReports  | -       | PDF报表生成  |
| **模板引擎** | Thymeleaf      | -       | HTML模板渲染 |

### 1.2 核心技术特点

1. **前后端分离架构**：后端提供RESTful API，前端Vue独立部署
2. **多端支持**：Web管理端 + 移动端（uni-app）
3. **RBAC权限控制**：基于角色的访问控制模型
4. **状态机控制**：流转卡工序状态流转控制
5. **双模打印策略**：JasperReports（PDF归档）+ Thymeleaf（网页预览）

## 三、核心功能代码实现详解

### 3.1 零件分流机制

**实现文件**: [TrialTaskProdServiceImpl.java](file:///d:/毕业设计/trial-admin/ruoyi-system/src/main/java/com/ruoyi/trial/service/impl/TrialTaskProdServiceImpl.java#L220-L311)

**核心逻辑**:

```java
@Override
@Transactional(rollbackFor = Exception.class)
public int forkTrialTaskProd(TrialTaskProd trialTaskProd, long oldTaskId) {
    // 1. 复制父母卡片的所有流转程序
    // 2. 设置关联任务ID（relatedTaskId）
    // 3. 处理分流卡片的状态：
    //    - 当前环节为正在填报(1)
    //    - 前面环节为已审核(3)
    //    - 后面的程序环节变更为未填报(0)
    // 4. 更新原卡片的分流数量
}
```

**状态流转规则**:

| 工序位置   | 状态值 | 含义   |
| ------ | --- | ---- |
| 当前环节之前 | 3   | 已审核  |
| 当前环节   | 1   | 正在填报 |
| 当前环节之后 | 0   | 未填报  |

### 3.2 程序变更/回退控制

**实现文件**: [TrialTaskProdServiceImpl.java](file:///d:/毕业设计/trial-admin/ruoyi-system/src/main/java/com/ruoyi/trial/service/impl/TrialTaskProdServiceImpl.java#L359-L382)

**核心逻辑**:

```java
@Override
public int flowTrialTaskProd(TrialTaskProd trialTaskProd, String userId) {
    // 1. 校验任务状态（已结束的任务无法变更）
    // 2. 更新当前程序序号和名称
    // 3. 重新计算所有工序状态：
    //    - 当前环节之前 → 已审核
    //    - 当前环节 → 正在填报
    //    - 当前环节之后 → 未填报
    // 4. 记录操作日志（actionType=2表示变更操作）
}
```

### 3.3 状态机控制

**工序状态定义**:

| 状态码 | 含义   | 可流转目标                 |
| --- | ---- | --------------------- |
| 0   | 未填报  | → 正在填报(1)             |
| 1   | 正在填报 | → 待审核(2)              |
| 2   | 待审核  | → 已审核(3) 或 回退到正在填报(1) |
| 3   | 已审核  | 终态                    |

| MyBatis拦截器 | [OptimisticLockInterceptor.java](file:///d:/毕业设计/trial-admin/ruoyi-framework/src/main/java/com/ruoyi/framework/interceptor/OptimisticLockInterceptor.java) |

| 单元测试 | [OptimisticLockTest.java](file:///d:/毕业设计/trial-admin/ruoyi-system/src/test/java/com/ruoyi/trial/service/impl/OptimisticLockTest.java) |

**实现原理**:
```java
// 1. 实体类添加version字段
private Integer version;

// 2. MyBatis拦截器自动在UPDATE语句添加版本条件
UPDATE trial_task_prod SET ..., version = version + 1 
WHERE task_id = ? AND version = ?

// 3. 更新失败时抛出乐观锁异常
if (updateCount == 0) {
    throw new OptimisticLockException("数据已被其他用户修改，请刷新后重试");
}
```

**应用场景**:
- 流转卡信息更新
- 工序详情填报
- 审核操作
