 1.2.1 国内研究现状
 近年来，国内汽车制造企业在数字化转型方面取得了显著进展，特别是在生产制造执行系统（MES）领域。然而，针对新产品试制环节的信息化系统研究相对较少。传统的纸质流转卡模式在试制过程中暴露出效率低、易出错、追溯难等问题。国内学者和企业开始探索电子化流转卡系统，主要集中在以下几个方面：

 - **MES系统在量产领域的应用**：国内大型汽车企业如上汽、一汽等已广泛应用MES系统进行量产管理，但这些系统在试制场景下存在灵活性不足、配置复杂等问题。
 - **轻量级信息化系统研究**：针对试制多品种、小批量、高频变更的特点，国内研究开始关注轻量级、易配置的信息化解决方案。
 - **移动采集技术应用**：随着智能手机和工业平板的普及，基于移动终端的扫码采集技术在试制现场得到应用，提高了数据采集的实时性和准确性。
 - **二维码技术集成**：二维码作为信息载体，在试制过程中的应用逐渐增多，用于标识零件、工序和流转状态。

 1.2.2 国外研究现状
 国外汽车制造企业在试制信息化方面起步较早，主要研究集中在：

 - **柔性制造系统（FMS）**：国外学者提出的柔性制造系统理论为试制信息化提供了理论基础，强调系统的灵活性和快速响应能力。
 - **数字孪生技术**：数字孪生技术在汽车试制中的应用，实现了物理世界与数字世界的实时映射。
 - **云服务与边缘计算**：利用云平台和边缘计算技术，实现试制数据的实时处理和分析。
 - **人工智能应用**：在质量预测、缺陷检测等方面应用AI技术，提高试制过程的智能化水平。

 1.3 主要研究内容 
 本研究针对汽车制造企业试制环节的痛点，设计并实现了一套基于JAVA的试制信息化管理系统。主要研究内容包括：

 1. **系统架构设计**：采用前后端分离架构，结合Web管理端和uni-app移动端，实现多端协同工作。
 2. **核心业务逻辑实现**：
    - 零件分流机制：实现一码多卡生成逻辑，支持试制过程中的零件分支管理
    - 程序变更/回退控制：基于状态机的工序流转管理，支持工序回退和状态重置
    - 扫码自动定位：利用二维码技术实现工序的快速定位和状态识别
 3. **并发控制机制**：基于MyBatis拦截器的乐观锁实现，解决多用户同时操作的并发冲突问题。
 4. **双模打印策略**：结合JasperReports和Thymeleaf，实现高精度PDF归档和网页即时预览。
 5. **超期预警系统**：基于定时任务的超期预警机制，及时提醒相关人员处理超期任务。
 6. **安全性设计**：实现RBAC权限控制和JWT令牌认证，确保系统安全。

## 一、项目技术特点总结

### 1.1 技术栈概览

| 层次       | 技术             | 版本      | 说明       |
| -------- | -------------- | ------- | -------- |
| **后端框架** | Spring Boot    | 2.5.15  | 核心框架     |
| **持久层**  | MyBatis        | -       | ORM框架    |
| **数据库**  | MySQL          | -       | 关系型数据库   |
| **缓存**   | Redis          | -       | 缓存加速     |
| **前端框架** | Vue.js         | 2.6.12  | Web管理端   |
| **移动端**  | uni-app + Vue3 | 3.0.0   | 跨平台移动应用  |
| **UI组件** | Element UI     | 2.15.13 | UI组件库    |
| **认证**   | JWT            | 0.9.1   | Token认证  |
| **二维码**  | Google ZXing   | 3.5.1   | 二维码生成与解析 |
| **报表**   | JasperReports  | -       | PDF报表生成  |
| **模板引擎** | Thymeleaf      | -       | HTML模板渲染 |

### 1.2 核心技术特点

1. **前后端分离架构**：后端提供RESTful API，前端Vue独立部署
2. **多端支持**：Web管理端 + 移动端（uni-app）
3. **RBAC权限控制**：基于角色的访问控制模型
4. **状态机控制**：流转卡工序状态流转控制
5. **双模打印策略**：JasperReports（PDF归档）+ Thymeleaf（网页预览）
