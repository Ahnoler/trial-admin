# 乐观锁并发控制实现文档

## 一、概述

本项目实现了基于版本号（Version）的乐观锁并发控制机制，用于防止多人同时操作同一数据时的覆盖问题。

## 二、实现原理

### 2.1 乐观锁机制

乐观锁通过在数据表中添加 `version` 字段实现：

1. **读取数据时**：将 `version` 字段一起读出
2. **更新数据时**：
   - SQL 语句中添加 `WHERE version = ?` 条件
   - 同时将 `version` 字段值加 1
3. **检查更新结果**：
   - 如果更新影响行数为 1，说明更新成功
   - 如果更新影响行数为 0，说明数据已被其他用户修改，抛出异常

### 2.2 工作流程

```
用户A读取数据 (version=0) ──┐
                           │
用户B读取数据 (version=0) ──┤
                           │
用户A提交更新 (version=0→1) ─┼── 成功
                           │
用户B提交更新 (version=0→1) ─┴── 失败（version已变为1，条件不匹配）
```

```java
@Data
public class TrialTaskProd extends BaseEntity {
    // ... 其他字段
    
    /**
     * 乐观锁版本号
     */
```

### 3.3 Mapper XML 配置

```xml
<!-- 查询时包含version字段 -->
<sql id="selectTrialTaskProdVo">
    select task_id, ..., version
    from trial_task_prod
</sql>

<!-- 插入时初始化version为0 -->
<insert id="insertTrialTaskProd">
    insert into trial_task_prod (..., version)
    values (..., 0)
</insert>

<!-- 更新时version自增 -->
<update id="updateTrialTaskProd">
    update trial_task_prod
    SET ..., version = version + 1
    where task_id = #{taskId}
</update>
```

### 3.4 Service层处理

```java
@Override
public int updateTrialTaskProd(TrialTaskProd trialTaskProd) {
    trialTaskProd.setUpdateTime(DateUtils.getNowDate());
    int rows = trialTaskProdMapper.updateTrialTaskProd(trialTaskProd);
    if (rows == 0) {
        throw new OptimisticLockException("数据已被其他用户修改，请刷新后重试");
    }
    return rows;
}
```

## 四、数据库迁移

执行以下SQL脚本添加version字段：

```sql
-- 为 trial_task_prod 表添加 version 字段
ALTER TABLE `trial_task_prod` 
ADD COLUMN `version` INT(11) NOT NULL DEFAULT 0 COMMENT '乐观锁版本号';

-- 为 trial_task_detail_prod 表添加 version 字段
ALTER TABLE `trial_task_detail_prod` 
ADD COLUMN `version` INT(11) NOT NULL DEFAULT 0 COMMENT '乐观锁版本号';

-- 初始化现有数据
UPDATE `trial_task_prod` SET `version` = 0 WHERE `version` IS NULL;
UPDATE `trial_task_detail_prod` SET `version` = 0 WHERE `version` IS NULL;
```

完整脚本位于：[optimistic_lock_migration.sql](file:///d:/毕业设计/trial-admin/sql/optimistic_lock_migration.sql)

## 五、使用示例

### 5.1 正常更新流程

```java
// 1. 查询数据（包含version）
TrialTaskProd task = trialTaskProdService.selectTrialTaskProdByTaskId(1L);
// task.getVersion() = 0

// 2. 修改数据
task.setTitle("新标题");

// 3. 提交更新
trialTaskProdService.updateTrialTaskProd(task);
// 成功：version变为1
```

### 5.2 并发冲突处理

```java
try {
    trialTaskProdService.updateTrialTaskProd(task);
} catch (OptimisticLockException e) {
    // 数据已被其他用户修改
    // 提示用户刷新页面重新获取最新数据
    return AjaxResult.error("数据已被其他用户修改，请刷新后重试");
}
```

## 六、前端处理建议

### 6.1 错误提示

前端应捕获乐观锁异常，提示用户刷新：

```javascript
// Vue示例
async updateTask(task) {
  try {
    const res = await updateTrialTaskProd(task);
    if (res.code === 200) {
      this.$message.success('保存成功');
    }
  } catch (error) {
    if (error.message.includes('已被其他用户修改')) {
      this.$confirm('数据已被其他用户修改，是否刷新页面获取最新数据？', '提示', {
        confirmButtonText: '刷新',
        cancelButtonText: '取消',
        type: 'warning'
      }).then(() => {
        location.reload();
      });
    }
  }
}
```

### 6.2 数据刷新机制

在提交前重新获取最新数据：

```javascript
async function safeUpdate(taskId, updateData) {
  // 1. 获取最新数据（包含最新version）
  const latestData = await getTrialTaskProd(taskId);
  
  // 2. 合并更新数据
  const mergedData = { ...latestData, ...updateData };
  
  // 3. 提交更新
  return await updateTrialTaskProd(mergedData);
}
```

## 八、注意事项

1. **version字段必须包含在查询中**：所有查询语句都需要包含version字段
2. **新增数据version初始化为0**：插入时必须设置version初始值
3. **更新时检查影响行数**：Service层必须检查更新结果
4. **事务回滚**：乐观锁异常会导致事务回滚
5. **前端提示**：需要给用户友好的错误提示

## 九、扩展建议

如需为其他表添加乐观锁支持：

1. 实体类添加 `Integer version` 字段
2. Mapper XML 添加 version 字段映射
3. 执行数据库迁移脚本添加字段
4. Service 层更新方法添加乐观锁检查
